题目描述

农夫约翰对牛棚里昏暗的灯光感到不满，刚刚安装了一个新吊灯。

新吊灯由 $N$ 个灯泡组成，这 $N$ 个灯泡围成一圈，编号为 $0\sim N−1$。

奶牛对这个新吊灯非常着迷，并且喜欢玩以下游戏：

对于第 $i$ 个灯泡，如果在 $T−1$ 时刻，它左侧的灯泡（当 $i>0$ 时，为第 $i−1$ 个灯泡；当 $i=0$ 时，为第 $N−1$ 个灯泡）是开着，那么在 $T$ 时刻，就切换这个灯泡的状态。

这个游戏将持续 $B$ 单位时间。

给定灯泡的初始状态，请确定在 $B$ 单位时间后，它们的最终状态。

输入格式

第一行包含两个整数 $N$ 和 $B$。

接下来 $N$ 行，按顺序描述每个灯泡的初始状态，每行包含一个整数 $1$ （表示开）或 $0$（表示关）。

输出格式

共 $N$ 行，按顺序每行输出一个灯泡的最终状态。

数据范围

$3\leq N\leq 16,$
$1\leq B\leq 10^{15}$

题目分析

观察数据范围可知，表示状态的二进制01串最多只有16位，因此所有的状态总数为 $2^{16} = 65536$，而变化次数 $B$ 远大于状态总数，根据鸽巢原理，这说明最坏情况下状态变化的过程中一定存在环，由于一个状态有且仅有一个后继，我们可以很容易的将环找出来，然后重复走环的过程可以被略去，进而可以求出最终状态。

如果不重复走循环，那么最终走的步数会小于等于 $2^{16}$。

我们将用一个 int 类型的整数来表示一个二进制01状态，如果用 string 类型来存储状态，每次计算需要遍历一次字符串，不如 int 型来的快。而且进行状态间的逐位计算可以直接使用位操作符。

关于求环的长度，可以开一个数组 p[state]，记录每一个状态是第几步走到的，这样当两次走到同样一个状态点时，就可以拿当前的步数减去之前记录的步数，这样即求得环的长度（环的长度定义为构成环的点的个数）。
