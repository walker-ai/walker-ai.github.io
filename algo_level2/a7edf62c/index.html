树的最长路径

发表于2022-03-13|更新于2022-03-16|算法学习算法提高动态规划
|字数总计:1.6k|阅读时长:6分钟|阅读量:|评论数:

原题链接

题目描述

给定一棵树，树中包含 $n$ 个结点（编号 $1\sim n$）和 $n − 1$ 条无向边，每条边都有一个权值。

现在请你找到树中的一条最长路径。

换句话说，要找到一条路径，使得使得路径两端的点的距离最远。

注意：路径中可以只包含一个点。

输入格式

第一行包含整数 $n$。

接下来 $n−1$ 行，每行包含三个整数 $a_i, b_i, c_i$，表示点 $a_i$ 和 $b_i$ 之间存在一条权值为 $c_i$ 的边。

输出格式

输出一个整数，表示树的最长路径的长度。

数据范围

$1\leq n\leq 10000,$
$1\leq a_i, b_i\leq n,$
$−10^5\leq c_i\leq 10^5$

输出样例

前置知识

  • 简单路径：路径上的所有点至多只访问了一次

  • 树的直径：树上最长的简单路径

在分析这道题之前，我们先来看看怎样寻找一颗 树的直径。

当树的所有边权为正时

1、先任取一点作为起点，然后 DFS 或 BFS一下，找到距离该点最远的一个点 $u$。

2、再从点 $u$ 出发，DFS 或 BFS 一下。再找到距离 $u$ 最远的一点 $v$。

最后 $u$ 和 $v$ 之间的路径就是树的直径

为什么这样做是对的？

证明：

任取一点 $a$，搜索距离它最远的点 $u$。

关于两次DFS求树的直径的例题：

Longest path in a tree SPOJ 1437

题目分析

由于树边的权值可能是负数，故不能用上述的经典做法来求树的直径，需要用到树形 DP。

状态表示：

集合：所有以当前点为路径中最高点的树的路径构成的集合

属性 (Max)：集合中所包含的最大路径长度

状态计算：

集合划分：

以其 子树中的某个节点 作为 起点，以他作为 终点 的 粉色路径
以其 子树中的某个节点 作为 起点，以 子树中的某个节点 作为 终点 的 蓝色路径
以其 子树中的某个节点 作为 起点，以 非其子树的节点 作为 终点 的 橙色路径

对于第 $1$ 种情况，我们可以直接递归处理其子树，找出到当前子树根节点最长的路径长度即可

对于第 $2$ 种情况，我们在处理第 $1$ 种情况时，顺便找出 $1$ 类路径的 次长路径

把 最长 和 次长 拼在一起，就是我们要的第 $2$ 种情况

而对于第 $3$ 种情况，我们可以把它归类为其 祖先节点 的第 $1$, $2$ 种情况，让其 祖先节点 去处理即可

以上来自：彩色铅笔

即可这样表示：

1、所有以当前点为路径最高点，且当前点作为路径的一个端点的路径集合：找到其儿子的路径最值，并加上这条边 $$f[son] + w[son \rightarrow u]$$

2、所有以当前点为路径最高点，且穿过当前点的路径集合：找到所有儿子中路径最大值和次大值，分别加边再求和 $$\max \Big \lbrace f[son_i] + w[son_i \rightarrow u]) + \max_{sub} \Big \lbrace f[son_j] + w[son_j \rightarrow u]) \Big \rbrace$$

TIPS:

这里有必要说明一下为什么要建双向边。

首先我们知道，如果从一颗树的根节点开始往下DFS，是一定能够搜到树的直径的，但是问题是本题中的树是一颗无根树，没有特定的根节点，所以初始时我们并不知道它的拓扑结构是什么样的。但我们可以任意指定一个节点为“根节点”来进行DFS，这样它的拓扑结构就唯一确定下来了，并在搜索过程中设置 if (j == father) continue 来防止它往上搜索。
