踩坑记录

顺序与重要性无关

git区分大小写问题

默认情况下，git commit不区分大小写，这也就是导致了我这个音乐界面为什么本地能够显示，push上去就显示不了的原因之一，以后在创建仓库的时候记得一定要设置为区分大小写，否则到最后遇到什么问题你可能死都不知道怎么死的。

新标签页创建的问题

从某网友上copy下来的关于aplayer的js和css代码其实是使用了jsdeliver的cdn，如果要用的话最好下到本地使用，路径规则与Tags，Categories下的img一致。另外，这里我卡了很长时间，大概三四个小时，我以为是css和js文件解析的问题，于是一直在找相关的解决方案，但最后我发现甚至连music这个界面在github page上都是显示404，于是我排除了所有的aplayer文件和meting文件，发现仍然404，这说明我这个music界面压根就是无效的，当场晕倒…原来整个解决问题的方向就是错的，但令我困惑的是这个界面的创建实在是没有什么坑点，最后回想可能是我当初这个music文件夹是手动创建的…正确的姿势是在博客根目录下执行 hexo new page "music"，然后在 /music 目录下编辑 index.md。

播放器的问题

网易云（netease）对blog的内嵌音乐支持的是比较好的，其官方提供了生成链接，如果你想为自己的blog添加背景音乐，甚至循环播放列表，那么你可以使用自己的网易云账号，在你的账号中创建blog歌单，然后添加你喜欢的歌曲进去，最后用官方提供的生成链接，取出ID加到meting插件的js中，至于什么是meting插件和aplayer插件，我只能说他们是一种支持html5的网页内嵌播放器插件，至于如何使用，请出门百度查阅官方文档。这里比较麻烦的是aplayer插件、meting插件和hexo-tag-aplayer插件的部署，并且网上很多教程都没有用，最后通过这里解决了问题。

http和https协议的问题

在我们使用hexo + github搭建博客时，有时会让我们填写自己github地址或者生成环境仓库的url，使用http还是https是有极大区别的，弄不好会十分麻烦，具体什么区别我也不知道，反正遇到时留意些就行了。

评论系统

这里的评论系统我使用的是valine，需要自己去LeanCloud官网注册一个账号，然后白嫖一个应用，然后创建生成环境，将应用部署到 https://github.com/sviptzk/Valine-Admin-Server 中，（Valine貌似听说在某个版本以后就停止维护了，原因是因为有暴露IP和邮箱地址的风险？Valine-Admin是某大佬基于Valine二次开发的一款应用，提供了许多额外功能例如游客在某站主的blog下评论了一些信息，站主可以及时在预设的邮箱中收到提醒信息，游客也能及时收到站主的回复消息，这些功能都是基于Valine-Admin以及leancloud的），但由于leancloud提供的免费应用功能是十分有限的，免费应用一天必须休眠六小时，想让其合理工作的话需要预设定时唤醒任务，具体百度。

聊天系统

blog page的右边栏内置了一个聊天系统的功能，我是用的是Chatra，这玩意没什么好说的，直接在框架内部根据文档提示配置就行了，不过这个东西消息提示音还挺好听的。

文章路径问题

文章路径一定要归档处理好，路径与站点配置文件.config.yml中的URL有关，可以用你自己喜欢的变量，例如我可以把permalink设置为 mypath，然后在每篇文章的Front-Matter自定义变量mypath为自己想要该文章存储的路径，为了方便，可以在scaffolds中设置初始Front-Matter，在模板中新增mypath变量。而默认的文章路径是:year/:month/:day/:title，这样如果你的文章标题含中文，那么它最后解析出来的URL就会是一串乱码，不仅不美观而且还会引发一些未知的bug，不利于SEO。

更新：

上面这样设置路径是不对的，这会使得每篇文章的URL不唯一，无法索引到具体地址，这里需要使用hexo-abbrlink插件，具体怎么使用查文档，有一点需要注意就是有可能会出现 abbrlink-undefined，遇到这种问题可能有几种情况：

1、文章缓存没有清除，直接 hexo clean && hexo g 即可

2、layout引起的问题，如果你的文章开头Front-Matter有设置布局选项layout: [layout]，那么就无法生成abbrlink，具体原因我还不知道，但还好我这里还没有生成过多文章。以后生成新文章记得直接执行 hexo new "xxx"，这样就不会指定布局，但是缺点是所有布局都是统一的。这里参考自Cycc。

另外注意：

一定要在最开始就设置好abbrlink的算法等相关设置，否则后面再改动就会使得原来链接无效，该页面上的评论等内容会全部丢失，切记！

关于本地Blog文件夹迁移的问题

近几天搞一个服创项目，前后端都要在本地跑起来，而且要求的 nodejs 版本还比较旧，我电脑上的 nodejs 版本太高了，但由于经常要更Blog，所以就下了一个 nvm 用来管理 nodejs 的多个版本，可这玩意恶心的很，要先卸载 nodejs，卸不干净还不能安装，于是我就把 nodejs 全卸了，好家伙，博客环境全没了，但好在迁移博客文件夹需要改动的地方不是很多。

首先原先的文件夹肯定是不能再用了，但配置文件全在里面，一定要备份，不备份就等着哭吧Blog从0开始搭2333，然后在一个新的路径下创建 Blog 文件夹，然后就是正常的搭 hexo 的过程，npm install hexo-cli，npm install，先把新博客搭起来，随后上主题网站上下载新主题，下载完之后将原先站点配置文件 _config.yml 复制进来，更改主题名，保持和下载的主题文件夹名一致。然后将原先根目录下的 hexo-offline.config.cjs （好像是跟 cdn 配置有关的文件）、package-lock.json、package.json （所需要的模块版本各种信息，有了它你就可以将之前所有的依赖包全部装回来）文件复制进来，然后将根目录下的 source 全部覆盖（这里面有你发布的文章的所有源文件），themes 目录下的 source 文件夹全部覆盖，以及 themes 目录下的主题配置文件 _config.yml 全部覆盖。根目录下的 image 全部覆盖（部分图片在这里保存），最后就是发布模板 scaffolds 下文件全覆盖。这样，就可以原封不动的将博客的文件夹全部迁移到新的路径下了。

另外还有一个地方需要改动，那就是像上面这样还是有一个东西没有加，那就是超链接丢失问题，需要按照上面记录的再操作一遍。
